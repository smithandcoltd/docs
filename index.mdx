The **OPBG Clients Portal** is the workflow layer between **JobTread** and the people using OPBG's project-finance processes.

In plain language:

- **JobTread** holds the source job, order, invoice, bill, and specification data.
- **The portal** controls what each user sees and which workflow they follow.
- **Supabase** stores the portal's own state, such as dashboards, draws, bills, user roles, share links, and settings.

This docs site is meant to answer two questions quickly:

1. What does this part of the app do?
2. Where does its data come from?

It is written for engineers first, but it should also be readable for operations, accounting, and product teammates.

## What this site covers

- who can access which routes and actions
- what the important construction-finance terms mean in this product
- how data moves between JobTread, Supabase, and the React app
- what each major page or tab is responsible for
- which tables, Edge Functions, and permissions matter in day-to-day work

## Primary source of truth

When documentation and code disagree, trust the code:

- Route behavior: `src/App.tsx`
- Workflow modes: `src/utils/workflowMode.ts`
- Page behavior: `src/pages/**` and `src/components/**`
- Fetching and mutations: `src/hooks/**`
- Integration logic: `supabase/functions/**`

## Recommended reading order

1. Start with **Roles and access** and **Portal modes**.
2. Read the concept pages if construction draw billing is new to you.
3. Read the core logic pages before changing workflow or finance behavior.
4. Use the feature pages when you are working in one part of the app.
5. Use the reference pages when you need a table, role, or integration lookup.

## Major product surfaces

| Surface | Purpose |
| --- | --- |
| Dashboard | The project-facing page for contract summary, change orders, specifications, and in 40-40-20 mode, invoices |
| Invoice Draw | The internal finance workspace for schedule review, draw packages, bills, and in 40-40-20 mode, invoices |
| Admin Panel | The admin-only area for users, archived dashboards, restricted cost codes, and task assignees |
| Viewer Home | The client landing page that shows only dashboards assigned to that client |
| Public Specifications | The no-login specifications page opened by a share link token |
