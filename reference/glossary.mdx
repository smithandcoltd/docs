---
title: "Glossary"
description: "Domain and application terms used throughout the OPBG Clients Portal"
---

Use this page when a feature page uses a term that sounds obvious but has a specific meaning in this product.

| Term | Meaning |
| --- | --- |
| Dashboard | Local portal record that connects a JobTread job to the app |
| Client | External viewer assigned to one or more dashboards |
| Superintendent | Internal non-admin role used for draw creation and project workflow |
| Draw | Internal package representing a billing request |
| Sign-off package | The 40-40-20 variant of the draw workflow |
| Statement of Values (SOV) | Contract-value breakdown used in standard draw mode |
| Hide financials | Dashboard mode that removes contractor-summary financials from the client-facing dashboard |
| 40-40-20 mode | Dashboard mode that switches the internal workflow to sign-off packages and changes client-facing tabs |
| Customer order | Client-side approved scope document in JobTread |
| Vendor order | Purchase order or commitment to a vendor |
| Vendor bill | Cost-side billing record tied to a vendor and ideally a PO |
| Customer invoice | Client-facing invoice in JobTread |
| Restricted cost code | Parent phase marked accounting-only, with inherited child restrictions |
| Vendor not let | Approved scope without an issued PO, shown as `VENDOR NOT LET` |
| Share link | Public token-backed link for specifications |
| Ready draw | Approved local draw package ready for downstream accounting steps |
| Waiver | Lien-waiver record tracked locally for vendor payment support |
